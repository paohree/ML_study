# 빅데이터분석기사 필기 — 시험 직전 핵심노트

## 1과목 — 빅데이터 분석 기획

### 빅데이터 특징
- 3V: Volume(양), Velocity(속도), Variety(다양성)
- 5V: 3V + Value(가치), **Veracity(신뢰성/정확성)**
  - Veracity = 데이터의 신뢰성, 정확성 (유용한 정보 추출이 아님)
- Variety = 정형 / 반정형(JSON, XML, 로그) / 비정형

### 하둡 / MapReduce
- HDFS: NameNode(메타데이터) + DataNode(실제 데이터)
- MapReduce 흐름: Map → 셔플/소트 → Reduce
  - Map: 키-값 쌍으로 쪼개기
  - 셔플/소트: 같은 키끼리 모으기
  - Reduce: 집계
- Spark: MapReduce 대체, 메모리(RAM) 기반 → 빠름

### 데이터 수집 도구
- Sqoop: RDB → 하둡, 배치 수집
- Flume: 로그 파일 실시간 스트리밍 수집 (하둡으로 직접)
- Kafka: 생산자 → Kafka → 소비자, 범용 메시지 스트리밍 (중간 버퍼)

### NoSQL 종류
| DB | 방식 | 예 |
|---|---|---|
| Redis | 키-값 (캐시) | 세션 저장 |
| HBase | **분산 컬럼 DB** (분산 파일 시스템 아님!) | 대용량 컬럼 |
| Cassandra | 컬럼 기반 | 분산 환경 |
| MongoDB | 문서(JSON) 기반 | 비정형 |

### 분석 방법론
**CRISP-DM 6단계** (비선형 순환)
1. 비즈니스 이해
2. 데이터 이해
3. **데이터 준비** (전처리, 변수 선택, 파생변수 생성)
4. 모델링
5. 평가
6. 배포

**KDD 5단계**: 선택 → 전처리 → 변환 → 마이닝 → 해석/평가

**SEMMA**: 샘플(Sample) → 탐색(Explore) → 수정(Modify) → 모델(Model) → 평가(Assess)

**빅데이터 분석방법론 5단계** (CRISP-DM과 다른 별개 개념)
1. 분석 기획
2. 데이터 준비
3. 데이터 분석
4. 시스템 구현
5. 평가 및 전개

**분석 마스터 플랜 우선순위**: 전략적 중요도 / ROI / 실행 용이성

**분석 활용 계획**: 모델링 끝나고 세우는 게 아님 → **프로젝트 시작 단계부터** 수립

### ETL
- Extract(추출) → **Transform(변환, 정제)** → Load(적재)
- T = 변환/정제 (운송 아님!)

### 데이터 저장소
- 데이터 레이크: 원시 데이터 그대로 저장 (정형/비정형 모두)
- 데이터 웨어하우스(DW): 분석용, 정형 데이터
  - 4특성: **주제지향성 / 통합성 / 시계열성 / 비휘발성**
- 데이터마트: DW 일부, 특정 부서용

- OLTP: 실시간 트랜잭션 처리 (데이터 쌓기)
- OLAP: 쌓인 데이터 분석 (다차원 분석)

### 데이터 관리
- 메타데이터: 데이터에 대한 데이터
- MDM (Master Data Management): 핵심 데이터를 **하나의 기준으로 통일**
- 데이터 품질 4속성: **완전성 / 일관성 / 정확성 / 최신성**
- 데이터 무결성: 데이터가 정확하고 일관성 있게 유지

### 데이터 거버넌스 3요소 (IT인프라는 구성요소 아님)
- **원칙(Principle)**: 데이터 유지관리 지침/가이드
- **조직(Organization)**: 역할과 책임
- **프로세스(Process)**: 데이터 관리 활동과 체계

### 데이터 3법 / 비식별화
- 개인정보 보호법, 정보통신망법, 신용정보법
- **가명정보**: 재식별 가능, 동의 없이 연구/통계 목적 사용 가능
- **익명정보**: 재식별 불가, 개인정보보호법 적용 안 됨

비식별화 기술: 가명처리 / 총계처리 / 데이터 범주화 / 데이터 마스킹 / 랜덤라운딩 / 제어라운딩
- **데이터 마스킹**: 수준 높을수록 식별 어려워짐 (쉬워지는 게 아님)

### 변수의 척도
| 척도 | 설명 | 예 |
|---|---|---|
| 명목 | 분류만 (순서/크기 없음) | 혈액형, 성별 |
| 순서(서열) | 순서 있음, 간격 불일치 | 학점(A/B/C), 만족도 |
| 등간 | 간격 동일, 절대 0 없음 | 온도(섭씨), 점수 |
| 비율 | 절대 0 있음 | 키, 몸무게, 매출 |

## 2과목 — 빅데이터 탐색

### 결측치 메커니즘 (MCAR / MAR / MNAR)
- **MCAR** (완전 무작위 결측): 결측 원인이 어떤 변수와도 무관. 완전히 랜덤.
- **MAR** (무작위 결측): 결측 원인이 **다른 변수**에 있음. 결측값 자체와는 무관.
  - 예) 남성이 우울증 문항 잘 안 씀 → 원인은 성별(다른 변수)
- **MNAR** (비무작위 결측): **결측값 자체**가 결측의 원인.
  - 예) 연봉 높은 사람이 연봉 안 씀, 혈압 높을수록 혈압 미기록

### 결측치 처리
- 행 제거 / 평균 대체 / **중앙값** 대체 / 최빈값 대체 / KNN
- **이상치 있을 때 → 중앙값** (평균은 이상치에 민감)
- **범주형 결측치 → 최빈값**
- 회귀분석으로 대체 → 변수 간 관계 보존 (약해지는 게 아님)
- **다중 대치법**: 결측치를 여러 방식으로 다르게 채워 데이터셋 여러 개 생성 → 불확실성 반영
- **Hot-deck 대치**: 비슷한 관측치의 **실제값** 그대로 가져와 대치 (KNN은 평균, Hot-deck은 실제값)

### 이상치 탐지
- Z-score: |Z| > 3 이면 이상치, Z = (x - μ) / σ
- IQR: Q1 - 1.5×IQR 미만 또는 Q3 + 1.5×IQR 초과
  - IQR = Q3 - Q1
  - 범위(max-min)는 이상치에 영향받음, IQR은 영향 안 받음

### 정규화 vs 표준화
- **정규화(Min-Max)**: 0~1 범위 압축, 이상치 없을 때
- **표준화(Z-score)**: 평균 0, 표준편차 1, 이상치 있을 때
- **Box-Cox 변환**: **양수 데이터에만** 적용, 정규분포에 가깝게 변환

### 인코딩
- **원핫 인코딩**: 0/1 행렬, 순서 없는 범주형 (ex. 고양이/개 → [1,0] / [0,1])
- **레이블(Label) 인코딩**: 정수 변환, 순서 있는 경우에만
- **오디널(Ordinal) 인코딩**: 순서대로 정수 할당
- **Binary 인코딩**: 이진 코드로 표현, 원핫보다 학습 빠름

### 대표값
- 평균: 이상치에 가장 민감
- 중앙값: 이상치에 덜 민감
- 최빈값: 이상치에 영향 거의 없음, **연속형 데이터에는 부적합**

### 왜도 (Skewness)
- **왜도 = 0**: 대칭 분포
- **왜도 > 0** (양의 왜도, 오른쪽 꼬리): **최빈값 < 중앙값 < 평균**
- **왜도 < 0** (음의 왜도, 왼쪽 꼬리): 평균 < 중앙값 < 최빈값
- 첨도(Kurtosis): 분포의 뾰족한 정도

### 공분산행렬 vs 상관행렬
- **공분산**: 단위가 살아있는 상관관계. 단위가 **같을 때** 사용 (예: PCA에서 변수 단위 동일)
- **상관행렬**: 단위 제거하고 -1~1로 표준화. 단위가 **다를 때** 사용
- 변수 단위가 다르면 공분산 수치 자체가 의미없어짐 → 상관행렬로

### 변동계수 (CV)
- 표준편차 ÷ 평균
- 단위가 **다른** 자료를 비교할 때 사용

### 상관관계
| 계수 | 언제 쓰나 |
|---|---|
| **피어슨** | 등간/비율척도, 선형 관계, 정규분포 가정 |
| **스피어만** | **서열척도**, 이상치 있거나 비선형일 때 (순위 기반) |
| **포인트-비세리얼** | 이진변수 ↔ 연속/서열변수 |
| **크라메르의 V** | 범주형 변수 간 상관강도 |

- 상관계수: -1 ~ 1 (0이면 무관)
- **편상관분석**: 다른 변수 영향 통제 후 두 변수만의 순수 상관관계 측정
- **정준상관분석(CCA)**: 다수 독립변수 ↔ 다수 종속변수 상관관계

### 표본추출 방법
| 방법 | 그룹 내부 | 그룹 간 | 뽑는 방식 | 예시 |
|---|---|---|---|---|
| **층화추출** | **동질** (비슷한 애들) | 이질 | 각 층에서 일부 추출 | 성별/연령대로 나눠서 각각 추출 |
| **군집추출** | **이질** (다양한 애들) | 동질 | 집락 몇 개 통째로 추출 | 학교 몇 개 뽑아서 전수조사 |

⚠️ **층화=층 안 동질**, **군집=집락이 모집단 축소판(이질)**

### 점추정 4조건
| 조건 | 의미 | 키워드 |
|---|---|---|
| **불편성** | 추정량의 기댓값 = 모수 (치우침 없음) | 기댓값 = 모수 |
| **효율성** | 불편추정량 중 분산이 가장 작음 | 분산 최소 |
| **일치성** | 표본 크기 n↑ → 모수에 수렴 | n 커질수록 |
| **충분성** | 표본이 모수 추정에 필요한 정보를 최대한 활용 | 정보 최대 활용 |

⚠️ 헷갈리는 것: **효율 = 분산**, **충분 = 정보**

### 구간추정 / 신뢰구간
- 점추정(값 하나)보다 범위로 추정하는 게 더 정직함
- **95% 신뢰구간의 정확한 의미**: 같은 방법으로 구간을 100번 만들면 95번은 모수를 포함한다
  - ❌ 틀린 표현: "모수가 이 구간 안에 있을 확률이 95%다" (모수는 고정값, 확률 없음)
- 신뢰수준 높이면 (95%→99%) → 구간 **넓어짐**
- 샘플 크기 n 키우면 → 구간 **좁아짐** (표준오차 σ/√n 감소)

### 가설검정
- **귀무가설(H0)**: 차이/효과 없다
- **대립가설(H1)**: 차이/효과 있다
- **p-value**: 귀무가설이 참일 때 이 결과가 나올 확률
  - p < 0.05 → 귀무가설 기각 → 유의미한 차이 있음
  - p ≥ 0.05 → 귀무가설 기각 못 함
- 유의수준은 상황에 따라 90%, 95%, 99% 등 다양하게 선택 가능 (항상 95% 아님)

### 오류 구분
| | H0 사실 | H0 사실 아님 |
|---|---|---|
| H0 채택 | 옳은 결정 | **2종 오류** |
| H0 기각 | **1종 오류** | 옳은 결정 |

- **1종 오류(α)**: H0이 참인데 기각 → 효과 없는데 있다고 판단
- **2종 오류(β)**: H0이 거짓인데 채택 → 효과 있는데 없다고 판단

### 검정 종류
- **t검정**: 2집단 평균 차이. t = 평균차이 / 표준오차. 퍼짐 작은데 평균 차이 크면 → 진짜 차이
- **ANOVA(F검정)**: 3집단 이상 평균 차이. F = 집단 간 분산 / 집단 내 분산. t검정을 여러 번 하면 오류 누적되니까 한 번에 검정
- MANOVA: 독립변수 1개 이상 + **종속변수 여러 개**

### 범주형 검정 3종 (모두 카이제곱 검정)
| 검정 | 변수 수 | 상황 | 예시 |
|---|---|---|---|
| **적합도** | 1개 | 기대 분포와 비교 | 주사위가 균등하게 나오나? |
| **독립성** | 2개, 하나의 표본 | 두 변수 간 관계 | 성별 ↔ 흡연 여부 관련 있나? |
| **동질성** | 여러 집단, 각각 표본 | 집단 간 분포 같나? | 서울/부산/포항 지지 정당 분포 같나? |

### 비모수 검정 (정규분포 가정 불가할 때)
| 상황 | 모수 검정 | 비모수 검정 |
|---|---|---|
| 2집단 독립 비교 | t검정 | **Mann-Whitney U** |
| 같은 집단 전후 비교 | 대응표본 t검정 | **Wilcoxon signed-rank** |
| 3집단 이상 비교 | ANOVA | **Kruskal-Wallis** |

- 비모수 = 정규분포 가정 없이 **순위(rank)**로 비교
- Mann-Whitney U = Wilcoxon rank-sum (같은 검정, 이름만 두 개)
- 카이제곱 검정: 두 범주형 변수의 독립성
- **적합도 검정**: 데이터가 특정 분포를 따르는지 확인 (카이제곱 대표)
- 비모수 검정(윌콕슨 등): 정규분포 가정 없을 때

### 중심극한정리
- 표본 크기 **≥ 30**이면 모집단 분포와 관계없이 표본평균이 정규분포에 근사
- "표본 크기와 관계없이"라는 말이 나오면 틀린 것

### EDA
- 기초통계/그래프로 확인 가능: 결측치, 이상치, 데이터 분포
- **통계적 유의성은 기초통계로 확인 불가** → 가설검정 필요

### 파생변수
- 기존 데이터를 기반으로 새로운 인사이트를 얻기 위해 생성
- 수집 시점이 아닌 **데이터의 특성, 관계, 패턴** 반영이 일반적

### 불균형 데이터
- Accuracy만으로 평가 불가 (정상 95% / 스팸 5%인데 전부 정상 예측해도 95%)
- **SMOTE**: 소수 클래스 **오버샘플링** (언더샘플링 아님)
- 계층별(Stratified) k-fold로 클래스 비율 유지 가능

## 3과목 — 빅데이터 모델링

### 지도 vs 비지도 학습
- **지도학습**: 라벨(정답) 있음 → 분류, 회귀
- **비지도학습**: 라벨 없음 → 클러스터링, 차원축소

### 분류 vs 회귀
- **분류**: 카테고리 출력 (스팸/아님, 신용등급)
- **회귀**: 연속 숫자 출력 (집값, 전기사용량)

### 선형회귀
- 데이터에 가장 잘 맞는 직선을 찾아 예측
- 오차 기준: MSE 최소화
- 잔차 가정: 평균=0, 등분산, 정규분포
- **잔차 자유도 = n - p** (p=파라미터 수, 절편 포함, 항상 n-1이 아님)

### Lasso / Ridge / Elastic Net (과적합 방지 정규화)
| 방법 | 정규화 | 특징 |
|---|---|---|
| **Lasso (L1)** | 절댓값 합 최소화 | 일부 계수 → **0으로 만듦** (변수 선택 효과) |
| **Ridge (L2)** | 제곱합 최소화 | 계수를 0에 **가깝게** 축소 (0은 안 됨) |
| **Elastic Net** | L1 + L2 결합 | 두 방식의 장점 결합 |

### 로지스틱 회귀
- 이름에 '회귀'지만 **분류** 문제 사용
- 출력: 0~1 확률 → 0.5 기준으로 분류
- 활성화 함수: Sigmoid
- **odds** = p/(1-p) : 성공 확률 vs 실패 확률 비율
- **logit** = log(odds) : -∞~+∞ 범위로 변환 → 선형 방정식으로 모델링 가능
- 계수 β: 변수 1 증가 시 odds가 exp(β)배 변함 (선지 소거용)

### KNN (K-Nearest Neighbors)
- 새 데이터 → 가장 가까운 K개 이웃 찾기 → 분류(다수결) / 회귀(평균)
- **K 작으면 → 과적합**, K 크면 → 과소적합
- **Lazy learning**: 학습 단계 없음, 데이터 저장해뒀다가 예측 시 거리 계산
- 결측치 대치에도 같은 원리 사용 (비슷한 K개 찾아서 평균)

### SVM (Support Vector Machine)
- 마진(Margin)을 최대화하는 결정 경계 찾기
- **서포트 벡터**: 경계선에 가장 가까운 데이터 포인트
- 선형 + 비선형 분류 모두 가능 (커널 함수 사용)
- **초매개변수(C, 커널 매개변수) 최적화 필요** (필요 없다고 하면 틀린 것)

### 결정트리 (Decision Tree)
- 비모수적 방법 (정규분포 가정 불필요)
- 조건-결과(if-then)식 규칙 발견
- 동질성이 커지는 방향으로 분기
- 단점: 깊을수록 과적합
- 정지규칙: 최대 깊이 / 최소 샘플 수 / 정보이득 최소치

### 앙상블 방법
| 방법 | 학습 방식 | 대표 알고리즘 |
|---|---|---|
| **배깅** | 병렬 (독립적) | 랜덤포레스트 |
| **부스팅** | 순차 (오분류에 가중치) | AdaBoost, GBM, XGBoost, LightGBM |
| **스태킹** | 모델 결과를 또 다른 모델이 학습 | - |

- **AdaBoost**: 오분류 데이터에 더 큰 가중치 부여해 순차 학습
- **XGBoost**: GBM보다 **빠름** (병렬처리, 정규화 내장, 결측치 자동처리)
- **LightGBM**: leaf-wise 분기 (기존 level-wise와 다름)
- 배깅은 병렬화 가능, 부스팅은 순차라 병렬화 불가

### 나이브 베이즈
- 모든 특성이 **서로 독립**이라고 가정
- **사전확률 + 우도 → 사후확률** 계산 (우도를 계산하는 것이지 사후확률 아님)
- 텍스트 분류(스팸 필터)에 자주 사용

### k-fold 교차검증
- 데이터를 k개로 나눔 → k-1개 학습, 1개 검정 → k번 반복
- **홀드아웃보다 느림** (k번 반복 학습)
- k가 클수록 신뢰도 올라감 (k=10 > k=2)

### Bootstrap
- **복원추출**: 뽑고 다시 넣고 또 뽑음 → 같은 데이터 중복 가능
- 데이터가 적을 때 있는 데이터 재활용해서 샘플 여러 개 생성
- **0.632 Bootstrap**: 복원추출 시 원본의 약 63.2%만 뽑힘 → 나머지 36.8%를 테스트셋으로 사용

### 클러스터링 (비지도학습)
**K-Means**
- K개 중심점 랜덤 설정 → 가까운 것끼리 묶음 → 중심 재계산 → 반복
- 단점: K 미리 정해야 함, 초기값 민감, 원형 모양만 잘 잡음
- 최적 K 탐색: **엘보우 메소드(Elbow Method)** — 집단 내 제곱합 꺾이는 지점

**DBSCAN**
- 밀도 기반 클러스터링
- K 미리 안 정해도 됨, 복잡한 모양도 잡음

**계층적 군집분석 (Hierarchical)**
- K 미리 안 정해도 됨
- 결과: **덴드로그램** (트리 구조)으로 표현, 사용자가 잘라서 군집 수 결정

### 시계열 분석
**정상성 (Stationarity)**
- 정상 시계열: **평균/분산이 시간과 무관하게 일정**
- 비정상: 추세(trend)나 분산 변화가 있음
- 계절 패턴도 시점마다 평균이 달라지면 **비정상**

**차분 (Differencing)**
- 오늘값 - 어제값 → 추세 제거, 정상화
- 차분 후 그래프가 **위아래 일정 범위 안에서 진동, 방향성 없음** → 정상화 성공

**ADF 검정 (Augmented Dickey-Fuller)**
- 귀무가설: **"이 데이터는 비정상이다"**
- **p < 0.05 → 귀무가설 기각 → 정상** ← 직관과 반대! 주의
- p ≥ 0.05 → 비정상 → 차분 필요

**ARIMA(p, d, q)**
- **p** (AR 차수): 과거 실제값 몇 개 참고 → **PACF**로 결정
- **d** (차분 횟수): 정상화까지 차분 횟수
- **q** (MA 차수): 과거 예측 오차 몇 개 참고 → **ACF**로 결정
- 외우기: **P**ACF → **p**, ACF → q

**ACF vs PACF**
- **ACF** (자기상관함수): 현재 vs 과거 lag별 상관관계. **간접 영향 포함** → MA의 **q** 결정
- **PACF** (편자기상관함수): 중간 lag 영향 제거한 **순수 직접 상관관계** → AR의 **p** 결정
- ACF가 lag k에서 뚝 끊기면 q=k, PACF가 lag k에서 뚝 끊기면 p=k

### 연관분석
- 지지도(Support): A+B 같이 구매한 비율
- 신뢰도(Confidence): A 산 사람 중 B도 산 비율
- **향상도(Lift)**: 실제 연관성 (1보다 크면 연관 있음)
  - 향상도 = 신뢰도 / P(B)

### 차원 축소
**PCA (주성분 분석)**
- 첫 번째 주성분: **가장 큰 분산** 설명
- 이후 주성분: 앞 성분과 직교하면서 차례로 최대 분산 설명
- 주성분들 간 상관관계 없음

**SVD (특이값 분해)**
- 행렬을 특이벡터(U, V)와 특이값(Σ)으로 분해
- **정방행렬 아니어도** 적용 가능 (정방행렬만 된다고 하면 틀린 것)
- 차원 축소, 노이즈 제거에 활용

**K-medoids**
- K-Means와 동일한 구조, 단 중심점을 평균 대신 **실제 데이터 포인트**로 사용
- **이상치에 강함** (K-Means는 이상치에 중심점이 끌림)

**GMM (Gaussian Mixture Model)**
- 데이터가 여러 정규분포의 혼합이라고 가정
- **확률적 할당**: "클러스터 A에 속할 확률 70%, B 30%" (K-Means는 딱 잘라 할당)

**SOM (자기조직화지도)**
- 비지도학습
- 고차원 데이터를 저차원(주로 2차원) 그리드에 매핑 (시각화 목적)

### 딥러닝
- 인공신경망: 입력층 → 은닉층(여러 겹) → 출력층
- 순전파: 입력 넣어서 예측
- 역전파: 오차를 뒤에서 앞으로 전달하며 **편미분**으로 가중치 조정
- 학습률(Learning Rate): 가중치 조정 폭

**활성화 함수**
- **ReLU**: 0보다 작으면 0, 크면 그대로 (은닉층에서 주로 사용)
- **Sigmoid**: 0~1 출력, 이진 분류 출력층
- **Softmax**: 합이 1, 다중 분류 출력층

**과적합 방지**
- 드롭아웃(Dropout): 학습 중 랜덤으로 노드 꺼버림
- 조기 종료(Early Stopping), 데이터 증강, 층/노드 수 줄이기

**NLP 모델 구분**
- **BERT**: 양방향, 문장 앞뒤 문맥 모두 고려
- **GPT**: 대용량 텍스트 사전학습 후 fine-tuning
- **Word2Vec**: 단어를 벡터로 변환
- **YOLO**: NLP 아님! 실시간 **객체탐지(이미지)** 알고리즘

**Transformer 구성요소**
- Self-Attention, Multi-Head Attention, Positional Encoding
- **Forget Gate는 LSTM 구성요소** (Transformer 아님)

**Seq2seq**
- 인코더: 입력 시퀀스 → 컨텍스트 벡터
- 디코더: 컨텍스트 벡터 → 출력 시퀀스

## 4과목 — 빅데이터 결과 해석

### 혼동행렬 (Confusion Matrix)
|  | 예측 Positive | 예측 Negative |
|---|---|---|
| 실제 Positive | TP | FN |
| 실제 Negative | FP | TN |

- **T/F** = 예측이 맞냐 틀리냐
- **P/N** = 뭐라고 예측했냐

### 분류 평가지표
- **Accuracy(정확도)** = (TP+TN) / 전체 — 불균형 데이터에 신뢰 못함
- **Recall(재현율/민감도)** = TP / (TP+FN) — 실제 양성 중 잡아낸 비율
- **Precision(정밀도)** = TP / (TP+FP) — 양성 예측 중 진짜 양성 비율
- **F1-Score** = 2 × (Precision × Recall) / (Precision + Recall) — 조화평균
- **Specificity(특이도)** = TN / (TN+FP) — 실제 음성 중 음성 예측 비율

### 상황별 지표 선택
- 암 진단 (놓치면 안 됨) → **Recall** 중시
- 스팸 필터 (정상 메일 스팸 처리 안 됨) → **Precision** 중시

### 임계값(Threshold) 트레이드오프
- 임계값 올리면 → 까다롭게 찍음 → **Precision ↑, Recall ↓**
- 임계값 내리면 → 많이 찍음 → **Precision ↓, Recall ↑**

### ROC 곡선 & AUC
- **X축: 1-특이도 (FPR)** — "X축=특이도"라고 하면 틀린 것
- **Y축: 민감도/Recall (TPR)**
- AUC = ROC 곡선 아래 면적
  - AUC = 1 → 완벽한 모델
  - AUC = 0.5 → 랜덤 수준 (동전 던지기)

### 회귀 평가지표
- **MAE**: 오차 절댓값 평균 — 이상치 덜 민감
- **MSE**: 오차 제곱 평균 — 이상치에 가장 민감
- **RMSE**: MSE의 제곱근 — 단위 복원
- **R²(결정계수)**: 모델이 데이터 변동을 설명하는 비율
  - R² = 1 → 완벽, R² = 0 → 평균값만 출력하는 것과 같음
- **F1 score는 분류 전용** — 회귀 모델(전기사용량 예측 등)에 사용 불가

### 시각화 종류
| 목적 | 그래프 |
|---|---|
| 연속형 데이터 분포 | 히스토그램 |
| 분포 + 이상치 확인 | 박스플롯 |
| 두 변수 관계 | 산점도 |
| 시계열 추이 | 선그래프 |
| 범주형 데이터 비교 | 막대그래프, 원형 차트 |
| 상관관계 행렬 | 히트맵 |
| **범주형 비율 + 상호작용** | **모자이크 플롯** |
| 여러 변수 동시 비교 | **레이더 차트(방사형)** |
| 3변수 표현 (x, y + 크기) | **버블 차트** |
| 여러 변수 특징 한눈에 | **비교시각화** |

- **히스토그램은 범주형 시각화가 아님** — 연속형 데이터 전용
- 모자이크 플롯 ≠ 히스토그램 안에 히스토그램 (직사각형 크기+색상으로 비율/상호작용)

### K-fold 교차검증 vs 홀드아웃
- k-fold: 홀드아웃보다 **느림** (k번 반복 학습)
- k 클수록 신뢰도 높음

## 자주 틀리는 함정 모음

| 함정 | 정답 |
|---|---|
| X축=특이도 | X축=**1-특이도** |
| Lasso는 계수를 0에 가깝게만 | Lasso는 0으로 **만들어** 변수 선택 |
| XGBoost가 GBM보다 느리다 | XGBoost가 **빠름** |
| 계층적 군집은 K 미리 정해야 함 | K 미리 **안 정해도 됨** |
| 오른쪽 꼬리: 최빈값 > 평균 | 오른쪽 꼬리: 최빈값 **<** 중앙값 **<** 평균 |
| YOLO = NLP | YOLO = **객체탐지(이미지)** |
| ADF p<0.05 → 비정상 | p<0.05 → 귀무가설("비정상이다") 기각 → **정상** |
| MNAR = 다른 변수가 원인 | 다른 변수 원인 = **MAR**, 결측값 자체가 원인 = **MNAR** |
| Forget Gate = Transformer | Forget Gate = **LSTM** |
| 잔차 자유도 = n-1 | 잔차 자유도 = **n-p** |
| 히스토그램 = 범주형 시각화 | 히스토그램 = **연속형** 전용 |
| SVD = 정방행렬만 | SVD = **직사각형 행렬도** 가능 |
| ETL T = 운송 | ETL T = **변환/정제** |
| HBase = 분산 파일 시스템 | HBase = **분산 컬럼 DB** |
| 데이터 거버넌스에 IT인프라 포함 | IT인프라 **포함 안 됨** (원칙/조직/프로세스) |
| 표본평균은 크기 관계없이 정규분포 | 표본 크기 **≥ 30**일 때 (중심극한정리) |
| 나이브 베이즈 = 사전확률+사후확률로 우도 계산 | 사전확률+**우도**로 **사후확률** 계산 |
| F1 score = 회귀 평가 가능 | F1 score = **분류 전용** |
| 분석 활용 계획 = 모델링 후 수립 | **프로젝트 시작 단계부터** 수립 |
