## ★★★ 최우선 추가 필요

### SECI 모델 (암묵지↔형식지 변환 4단계)
- **공통화** (암→암): 경험 공유, 도제식 학습
- **표출화** (암→형): 언어로 표현, 문서화
- **연결화** (형→형): 문서끼리 조합/체계화
- **내면화** (형→암): 학습하여 체화

### 빅데이터 기능적 변화 4가지
- 사전처리 → **사후처리**
- 표본 → **전수**
- 질 → **양**
- 인과관계 → **상관관계**

### 빅데이터 위기요인 3가지 + 통제방안 (쌍 매칭 문제)
| 위기요인 | 통제방안 |
|---|---|
| 사생활 침해 | 동의→책임 |
| 책임원칙 훼손 | 결과기반 책임 |
| 데이터 오용 | 알고리즘 접근 허용 |

### 7V (빅데이터 특징 확장)
- 3V: Volume, Velocity, Variety
- 5V: + Value, Veracity
- **7V**: + Validity(타당성), Volatility(휘발성)

### YARN
- Yet Another Resource Negotiator
- 하둡 클러스터 **자원 관리** 담당

### 하둡 에코시스템 역할 표
| 도구 | 역할 |
|---|---|
| Hive | HiveQL 사용, DW 인프라 |
| Pig | 스크립트 기반 데이터 처리 |
| Impala | 실시간 SQL 쿼리 |
| **Tajo** | 대용량 DW + **ETL 수행** ← 기출 |
| Mahout | **머신러닝** |
| Oozie | 워크플로우 관리 |
| ZooKeeper | **분산 동기화** |

### 빅데이터 조직구조 3종 + 단점
| 구조 | 특징 | 단점 |
|---|---|---|
| **집중구조** | 별도 분석 조직 | 이원화 위험 |
| **기능중심구조** | 부서별 분석팀 | 전사 분석 어려움 |
| **분산구조** | 현업 배치 | 일관성 부족 |

## ★★ 중요 — 보강 권장

### DB 설계 5단계 순서
요구조건 분석 → 개념적 설계 → 논리적 설계 → 물리적 설계 → 구현

### DBMS 3종
- RDBMS (관계형)
- **OODBMS** (객체지향) ← 필수노트 누락
- NoSQL

### NoSQL 4가지 분류
| 유형 | 예시 |
|---|---|
| Key-Value | Redis |
| Column | HBase, Cassandra |
| Document | MongoDB |
| **Graph** | Neo4j ← 필수노트 누락 |

### 공공데이터 파일 형식
CSV, JSON, XML, XLS, XLSX, ZIP, TXT, HWP, JPG, MP4
- **SQL은 공공데이터 파일 형식 아님** ← 기출 함정

### CRISP-DM 비선형 강조
- 평가 단계에서 다시 비즈니스이해로 돌아갈 수 있음 ← 선지 함정

### EAI
- Enterprise Application Integration
- 기업 내 애플리케이션 간 데이터 통합 미들웨어

### 가명정보 결합 조건
- 가명정보끼리 결합은 **전문기관(결합전문기관)을 통해서만** 가능

### 데이터 사이언스 3영역
Analytics(분석) / IT(데이터관리) / 비즈니스컨설팅
