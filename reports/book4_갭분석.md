## ★★★ 최우선 추가

### 분석모형 평가 3가지 기준
- **일반화 가능성**: 새 데이터에도 성능 유지되는지
- **효율성**: 계산 비용/속도
- **예측 정확성**: 예측값 vs 실제값 오차

### 오분류율 (Error Rate)
- = 1 - Accuracy = (FP+FN) / 전체
- 낮을수록 좋음

### 모수 유의성 검정 (t-검정)
- 회귀계수 β가 유의한지: H₀: β = 0
- p값 < 0.05 → 해당 계수가 통계적으로 유의함

### Hosmer-Lemeshow 검정
- **로지스틱 회귀** 모형 적합도 평가
- **p > 0.05** → 모형이 적합함 (귀무가설 = 모형이 적합함)
- ⚠️ ADF와 반대 방향 — p > 0.05가 "좋음"

### 이익도표 / 이득도표
- **이득도표(Gain Chart)**: 누적 응답률 곡선
- **이익도표(Lift Chart)**: 상위 x% 타겟팅 시 전체 대비 효과 시각화
- Lift 개념 자체와 별도로 명칭 혼동 문제 출제

## ★★ 추가 권장

### 과대적합 방지 기법 6종
| 기법 | 설명 |
|---|---|
| **정규화(L1/L2)** | Lasso, Ridge |
| **드롭아웃** | 학습 중 일부 노드 무작위 제거 |
| **조기종료** | 검증 손실 올라가기 시작하면 학습 중단 |
| **배깅** | 부트스트랩 샘플로 분산 줄임 |
| **데이터 증강** | 훈련 데이터 인위적으로 늘림 |
| **교차검증** | 홀드아웃보다 과적합 탐지에 유리 |

### 교차검증 추가 유형
| 방법 | 설명 |
|---|---|
| **LOOCV** | k=n인 k-fold. 1개씩 빼고 학습 |
| **Stratified k-fold** | 클래스 비율 유지하며 분할 (불균형 데이터) |
| **Time Series Split** | 미래 데이터가 과거 학습에 안 들어가도록 분할 |

### 하이퍼파라미터 최적화 3종
| 방법 | 특징 |
|---|---|
| **Grid Search** | 모든 조합 탐색, 느림 |
| **Random Search** | 랜덤 샘플링, Grid보다 빠름 |
| **베이지안 최적화** | 이전 결과 반영, 가장 효율적 |

### PR 곡선 (Precision-Recall)
- 불균형 데이터에서 ROC보다 더 적합
- AUC-PR: PR 곡선 아래 면적, 1에 가까울수록 좋음

### 학습 곡선 / 검증 곡선
- **학습 곡선**: 훈련 데이터 크기 증가에 따른 성능 변화 → 과소/과대적합 진단
- **검증 곡선**: 하이퍼파라미터 값 변화에 따른 성능 변화

## 기존 항목 수정 필요

### Lift 위치 수정
- 현재 필수노트: 클러스터링 평가 섹션에 있음
- **수정**: Lift는 분류 모델(타겟 마케팅) 평가 지표 — 군집 평가 아님

### Cook's distance 기준값 추가
- **Cook's D > 1** (또는 > 4/n) → 영향력 있는 이상치로 판단

### 카파 상관계수 보완
- 다중 분류 문제에서 Accuracy 대신 사용하는 이유: 클래스 불균형 보정
