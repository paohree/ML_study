# 빅데이터분석기사 필기 — 빈출 우선순위 기반 GAP 분석

기준 날짜: 2026-03-26  
기준 시험일: 2026-04-04 필기

업데이트: PDF 4권 2차 정밀 판독 + `books/test1.pdf`, `books/test2.pdf` 실전형 문제 반영

이 문서는 `books/book1~4.pdf`, 현재 저장소의 `md` 노트, 최근까지 정리된 기출 GAP 문서를 함께 보고 만든 **시험 점수 중심 진단 문서**다.

목표는 교재 전범위 완독이 아니라:

- 실제 필기에서 자주 맞닥뜨리는 개념을 우선 채우기
- 함정 선지에 잘 낚이는 개념을 미리 정리하기
- 짧게 투자해서 점수화 가능한 계산형을 우선 확보하기

## 0. 결론 요약

현재 상태는 **핵심 개념은 이미 상당히 잡혀 있고, 합격권 진입 가능성은 높다.**  
다만 아직 **빈출 계산형 + 2/3과목 주변 개념 + 4과목 확장 평가지표**에서 구멍이 남아 있다.

즉, 문제는 "아무것도 모른다"가 아니다.

- 이미 잡은 것: 1과목 핵심, 2과목 기본 통계/전처리, 3과목 ML 기본 뼈대, 4과목 핵심 평가지표
- 아직 위험한 것: 계산형 손풀이, 선지 함정형 세부 개념, PDF에는 있는데 현재 노트에 덜 반영된 빈출 주변 주제

따라서 남은 기간 전략은 다음이 맞다.

1. **빈출 계산형을 먼저 고정**
2. **자주 나오는 주변 개념만 보강**
3. **교재 전범위를 다 외우려 하지 말고, 점수화 가능한 것부터 채우기**

추가 판단:

- 처음에는 교재 전체 중 일부만 선별해서 봤지만, 2차 판독 후에는 **교재의 후반부/주변부에도 실제 기출형 문제가 꽤 붙어 있다는 점**이 더 분명해졌다.
- 특히 **3과목 후반부(시계열, 범주형 자료 분석, 비모수 검정, 베이즈, 텍스트 마이닝)** 와 **4과목 중반부(잔차 분석, 교차 검증, 파라미터/하이퍼파라미터, 최종모형 선정)** 는 처음 생각보다 시험 대비 가치가 더 높다.
- 반대로 **도구 소개, 산업 전망, 시각화 디자인 원칙 세세한 설명** 은 여전히 후순위다.

### test1, test2가 추가로 보여준 것

- 시험은 "모든 내용을 깊게 파고드는 방식"보다 **넓은 범위의 기본 개념을 정확히 구분하는 방식**에 가깝다.
- 문제 유형은 대체로 아래 4개로 정리된다.
  - 정의/구분형: 옳은 것, 옳지 않은 것, 연결이 맞는 것 찾기
  - 계산형: 베이즈, Lift, RR/OR, z-score, PCA 기여율, 혼동행렬 지표
  - 적용형: 어떤 기법을 어떤 상황에 쓰는가
  - 함정형: 축/식/용어를 살짝 바꿔놓은 선지 소거
- 즉 합격 전략은 "전범위 완독"이 아니라 **빈출 범위의 정확도 확보 + 계산형 손풀이 + 자주 나오는 주변 개념 메우기**다.

### 너의 현실을 반영한 최종 해석

- 너는 이미 핵심 뼈대가 있어서, 쉬운 문제를 맞힐 기반은 충분하다.
- 현재 가장 큰 리스크는 "완전 생소한 초고난도 문제"가 아니라:
  - 3과목 후반부를 건너뛰어서 아는 문제를 놓치는 것
  - 계산형을 알고도 손으로 느리게 풀거나 실수하는 것
  - 4과목을 혼동행렬 위주로만 보고 확장 지표/진단을 비워두는 것이다.
- 따라서 지금부터는 **깊이보다 적중 범위와 정확도**를 우선해야 한다.

## 1. 우선순위 기준

이 문서에서 우선순위는 아래 기준으로 매겼다.

- **최우선**: 기출/후기/교재 구조상 반복 노출되고, 맞추면 점수 효율이 높은 것
- **중요**: 선지형 함정으로 자주 출제될 수 있고, 현재 노트에 빈 구멍이 있는 것
- **후순위**: 교재에는 있으나, 지금 시점에서 점수 효율이 낮은 것

표기 방식:

- `O` = 이미 비교적 안정적
- `△` = 개념은 아는데 아직 흔들림
- `X` = 현재 노트 기준 보강 필요

## 2. 과목별 현재 상태

### 1과목 전체 판단

- 핵심 빈출은 전반적으로 잘 잡혀 있음
- 지금 당장 급한 과목은 아님
- 다만 "교재에는 있는데 선지형으로 나올 수 있는 얕은 개념" 몇 개는 비어 있음
- `test1`, `test2` 기준으로 보면 HDFS, NoSQL, DW, CRISP-DM, 비식별화, 데이터 품질/거버넌스는 생각보다 직접적으로 물을 수 있음

현재 평가: **안정권에 가까움**

### 2과목 전체 판단

- 기본 점수원은 확보됨
- 그러나 확률/추정/검정/결측치 세부에서 빈 구멍이 남아 있음
- 계산형 대비가 아직 충분하지 않음
- `test1`, `test2` 기준으로는 전처리만 보는 과목이 아니라 **기초 통계/확률/추론을 넓게 보는 과목**에 가깝다

현재 평가: **기본은 됐지만 함정형 대비 부족**

### 3과목 전체 판단

- 여전히 가장 위험한 과목
- 지도/비지도 기본은 있으나, 시계열/범주형 분석/연관분석 계산/검증전략 같은 주변 주제가 덜 정리됨
- "보기 소거"가 안 되는 문제가 생길 가능성이 높음
- `test1`, `test2`를 보면 NLP, Transformer, Seq2Seq, SVD, SOM, Naive Bayes, ANOVA 해석까지 실제 문제형으로 충분히 붙는다

현재 평가: **핵심은 있으나 빈출 주변 범위 보강 필요**

### 4과목 전체 판단

- 혼동행렬, ROC/AUC, Precision/Recall 등 핵심은 강함
- 다만 회귀 평가 확장 지표, 군집 평가, 모형 진단 파트가 덜 정리됨
- `test1`, `test2` 기준으로는 시각화 함정 구분, Q-Q plot, 하이퍼파라미터, 엘보우, 평가 방식 구분도 무시하면 안 된다

현재 평가: **핵심 강점 과목, 확장 파트만 보강하면 좋음**

## 3. 최우선으로 채워야 할 것

이 섹션은 **시험 점수 효율이 가장 높은 항목**만 모아놓은 것이다.

### 3-1. 계산형

#### 1) 베이즈 정리

상태: `△`

왜 중요한가:

- 기출/변형 출제 반복
- 숫자만 바꿔서 내기 쉬움
- 맞추면 확실하게 점수화 가능

꼭 알아야 할 것:

- 사전확률 / 사후확률 / 가능도
- 전체확률의 법칙과 연결
- 표나 조건을 보고 식 세우기

현재 판단:

- 공식 자체는 알고 있음
- 직접 손으로 계산하는 숙련도는 더 필요

#### 2) 향상도(Lift)

상태: `△`

왜 중요한가:

- 연관분석에서 대표 계산형
- 개념과 계산을 같이 물어볼 수 있음

꼭 알아야 할 것:

- 지지도, 신뢰도, 향상도 차이
- 향상도 > 1 이면 양의 연관
- Gain/Lift 차트와 연결 가능

현재 판단:

- 개념은 있음
- 실전 계산 속도는 더 필요

#### 3) 혼동행렬 기반 지표 계산

상태: `O~△`

왜 중요한가:

- 4과목 핵심 점수원
- Accuracy, Precision, Recall, Specificity, F1, FPR가 한 세트로 나옴

꼭 알아야 할 것:

- TP / FP / FN / TN 구분
- Recall vs Precision vs Specificity
- ROC에서 FPR = 1 - Specificity

현재 판단:

- 기본은 잘 잡혀 있음
- 계산 속도와 실수 방지가 중요

#### 4) PCA 누적기여율 계산

상태: `△`

왜 중요한가:

- 기여율/누적비율 표를 주고 묻는 문제에 취약할 수 있음
- 2과목과 3과목 개념 연결 포인트

꼭 알아야 할 것:

- 특정 주성분 기여율 = 누적기여율 차이
- 누적기여율 기준 선택
- 고유값 1 기준, scree plot

현재 판단:

- 개념은 들었지만 아직 자동 반응 수준은 아님

#### 5) 회귀 ANOVA 표 해석

상태: `X`

왜 중요한가:

- 회귀 유의성, SSR/SSE/SST, F값 등 해석형 문제가 나올 수 있음
- 노트에는 일부 있으나 아직 실전 해석이 약함

꼭 알아야 할 것:

- SST = SSR + SSE
- F = MSR / MSE
- 자유도: 회귀 k, 잔차 n-k-1, 전체 n-1

현재 판단:

- 최우선 보강 필요

#### 6) RR / OR 계산

상태: `△`

왜 중요한가:

- 범주형 자료/분할표와 연결될 수 있음
- 개념형 + 계산형 혼합으로 출제 가능

꼭 알아야 할 것:

- 상대위험도 = 사건확률의 비
- 승산비 = odds의 비

현재 판단:

- 공식은 기억하지만 문제 적용 숙련 필요

#### 7) z-score / 표준화 점수 해석

상태: `△`

왜 중요한가:

- `test1`, `test2`에서 직접 계산형으로 붙기 좋은 유형이었다
- 평균, 표준편차, 상대적 위치 해석을 짧게 물을 수 있다

꼭 알아야 할 것:

- z = (x - 평균) / 표준편차
- z가 양수면 평균보다 큼
- 절댓값이 클수록 평균에서 멀다

현재 판단:

- 공식은 알 가능성이 높지만, 숫자를 넣고 바로 판단하는 연습은 더 필요

### 3-2. 개념형 최우선

#### 1) 과적합 / 과소적합 / 일반화 / 검증데이터 역할

상태: `△`

왜 중요한가:

- 모델링 전반에서 자주 나오는 공통 개념
- 2과목 불균형 데이터, 3과목 모델링, 4과목 진단에 걸쳐 연결됨

꼭 알아야 할 것:

- 학습 성능은 높은데 일반화가 나쁘면 과적합
- 학습 자체가 덜 되면 과소적합
- 검증 데이터는 하이퍼파라미터/모형 조정
- 테스트 데이터는 최종 평가용

현재 판단:

- 정의는 알지만, 보기형으로 꼬이면 흔들릴 가능성 있음

#### 2) MCAR / MAR / MNAR

상태: `X`

왜 중요한가:

- 2과목 결측치에서 교재 기준 중요
- 한 번 보면 쉬운데 안 보면 틀리기 쉬움

꼭 알아야 할 것:

- MCAR: 완전 무작위
- MAR: 다른 변수와 관련
- MNAR: 결측 자체와 관련

현재 판단:

- 현재 필수노트에 약함
- 짧게 외워서 점수화 가능한 항목

#### 3) 다중 대치법 / Hot-deck / nearest neighbor

상태: `X`

왜 중요한가:

- 결측치 처리 방법 구분형 문제에 적합
- 현재 노트에는 거의 없음

현재 판단:

- 최소 정의 구분은 채워야 함

#### 3-1) Box-Cox 변환

상태: `X`

왜 중요한가:

- `test1`, `test2` 기준으로 2과목 전처리/정규성 관련 문제로 직접 나올 수 있다
- "왜도 완화"와 연결해서 짧게 묻기 좋다

꼭 알아야 할 것:

- 분포를 정규성에 가깝게 만들기 위한 변환
- 분산 안정화, 왜도 완화

현재 판단:

- 현재 노트 반영이 약하다

#### 4) 공분산 행렬 vs 상관계수 행렬

상태: `X`

왜 중요한가:

- PCA 문제에서 바로 연결됨
- 선지형으로 내기 좋음

꼭 알아야 할 것:

- 단위 비슷하면 공분산 행렬
- 단위 많이 다르면 상관계수 행렬

#### 5) 시계열 기본

상태: `X`

왜 중요한가:

- 교재에서 분명히 비중 있게 다룸
- 최근 회차 변형형으로 내기 좋은 주제

꼭 알아야 할 것:

- 정상성
- AR / MA / ARIMA
- 차분
- ACF / PACF 기본 방향성

현재 판단:

- 현재 노트 공백이 큼
- 3과목 보강 1순위 중 하나

#### 6) 범주형 자료 분석: 적합도 / 독립성 / 동질성 검정

상태: `X`

왜 중요한가:

- "언제 어떤 검정을 쓰는가" 선지형으로 자주 낼 수 있음

꼭 알아야 할 것:

- 적합도 검정: 한 변수, 기대분포와 비교
- 독립성 검정: 두 변수 관계

#### 7) 비모수 검정 대표 4개

상태: `X`

#### 8) MAPE / MSLE / RMSLE / Adjusted R²

상태: `X`

왜 중요한가:

- 4과목 회귀 평가 확장 지표
- 현재 노트는 MAE/MSE/RMSE/R²까지는 좋지만 확장 지표가 비어 있음

현재 판단:

- 빠르게 메우면 효율 좋음

#### 9) 실루엣 점수

상태: `X`

왜 중요한가:

- 군집 수 선택 / 군집 품질 평가에서 대표 개념
- k-means와 연결되어 출제하기 좋음

꼭 알아야 할 것:

- -1 ~ 1 범위
- 1에 가까울수록 좋음

- [ ] Wilcoxon, Mann-Whitney, Kruskal-Wallis 정도는 구분할 수 있다
- [ ] K-medoids / GMM / SOM의 성격을 안다
- [ ] Transformer, BERT, GPT, Seq2Seq, Word2Vec의 차이를 대략 구분할 수 있다
- [ ] Naive Bayes와 로지스틱 회귀의 차이를 대략 설명할 수 있다

### 4과목 체크

- [ ] MAPE / MSLE / RMSLE / Adjusted R²를 구분할 수 있다
- [ ] 실루엣 점수의 범위와 해석을 안다
- [ ] ROC의 x축, y축을 정확히 안다
- [ ] Gain / Lift 개념을 안다
- [ ] 잔차 플롯, Q-Q plot, VIF, Cook's distance를 대략 해석할 수 있다
- [ ] 카토그램, 모자이크, 인포그래픽 유형을 보기에서 구분할 수 있다

### 1과목 체크

- [ ] DIKW를 안다
- [ ] 암묵지와 형식지를 구분할 수 있다
- [ ] 데이터베이스 특징 4개를 말할 수 있다

## 9. 마지막 판단

현재 paohree는:

- 필수 핵심은 이미 상당수 확보했고
- 노트 정리도 좋은 편이며
- 오답 흡수 속도도 빠르다

하지만 아직 시험 관점에서 남아 있는 리스크는 분명하다.

- **계산형을 손으로 푸는 속도**
- **3과목 후반부와 NLP/검정/시계열 공백**
- **4과목 확장 지표와 진단/시각화 공백**

따라서 남은 기간에는 교재 전체를 다시 도는 것보다, 이 문서의 `최우선`과 `중요` 항목만 압축 보강하는 게 가장 효율적이다.

`test1`, `test2`까지 반영해서 내리는 최종 판단은 더 명확하다.

- 이 시험은 "엄청 어렵고 깊은 한두 문제"보다 **넓은 범위의 기본 문제를 얼마나 안 틀리느냐**가 더 중요하다.
- 너는 이미 그 기본 뼈대는 갖고 있다.
- 그래서 합격을 가르는 건 새로운 거대한 범위를 공부하는 능력이 아니라, **3과목 후반 공백을 메우고 계산형을 손에 익히는 것**이다.

핵심 한 줄:

**지금부터는 "모든 내용 학습"이 아니라 "빈출 계산형 + 3과목 후반 + 4과목 확장/진단 + 선지 함정형 주변 개념"만 채우는 방식이 가장 합격 가능성을 높인다.**
