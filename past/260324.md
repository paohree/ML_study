# 260324 — 1과목/2과목 복습 + 3과목 전반

## 1과목 복습

### 빅데이터 3V
- Volume(양), Velocity(속도), Variety(다양성)
- Variety = 정형 / 반정형 / 비정형
- 반정형: 스키마는 없지만 구조는 있음 (JSON, XML, 로그파일)

### 하둡 / MapReduce
- 데이터가 너무 많아서 서버 한 대로 처리 불가 → 분산처리 필요 → 하둡 등장
- MapReduce 흐름: **Map → 셔플/소트 → Reduce**
  - Map: 데이터를 쪼개서 키-값 쌍으로 변환
  - 셔플/소트: 같은 키끼리 모음
  - Reduce: 같은 키끼리 집계
- 분산 저장소: HDFS (NameNode/DataNode)

### Spark
- MapReduce 대체 프레임워크
- MapReduce는 결과를 디스크에 저장 → 느림
- Spark는 메모리(RAM)에 올려서 처리 → 빠름

### 데이터 수집 도구
- **Sqoop**: RDB(MySQL 등)에서 하둡으로 배치 수집
- **Flume**: 로그 파일을 실시간 스트리밍으로 수집
- **Kafka**: 생산자 → Kafka → 소비자 구조의 메시지 큐 (중간 버퍼 역할)

### CRISP-DM 6단계
1. 비즈니스 이해
2. 데이터 이해
3. 데이터 준비 (전처리)
4. 모델링
5. 평가
6. 배포
- 비선형 순환 구조 (평가 후 다시 준비 단계로 돌아갈 수 있음)

### 데이터 3법 — 가명정보 vs 익명정보
- **가명정보**: 재식별 가능, 동의 없이 연구/통계 목적 사용 가능, 개인정보보호법 적용
- **익명정보**: 재식별 불가능, 개인정보보호법 적용 안 됨

## 2과목 복습

### 결측치 처리
- 대표값 대체: 평균, 중앙값, 최빈값
- 유사값 대체: KNN 기반
- 행 제거
- 이상치 있을 때 → **중앙값** 사용 (평균은 이상치에 민감)

### 이상치 탐지
- **Z-score**: |Z| > 3 이면 이상치
- **IQR**: Q1 - 1.5×IQR 미만, Q3 + 1.5×IQR 초과

### 정규화 vs 표준화
- **정규화(Min-Max)**: 0~1 범위로 압축, 이상치 없을 때 적합
- **표준화(Z-score)**: 평균 0, 표준편차 1로 변환 (범위 제한 없음), 이상치 있을 때 적합

### 인코딩
- **원핫 인코딩**: 카테고리를 0/1 행렬로 변환, 순서 없는 카테고리에 사용
- **레이블 인코딩**: 정수로 변환, 순서 있는 경우에만 사용 (순서 없는 데이터에 쓰면 모델이 크기 관계로 오해)

### 상관관계
- **피어슨**: 선형 관계, 정규분포 가정
- **스피어만**: 순위(rank) 기반, 이상치 있거나 정규분포 아닐 때 사용

### 가설검정
- **귀무가설**: 차이/효과가 없다는 기본 가정
- **대립가설**: 차이/효과가 있다는 주장
- **p-value**: 귀무가설이 참일 때, 이 결과가 나올 확률
  - p-value < 유의수준(0.05) → 귀무가설 기각 → 유의미한 결과
- **1종 오류**: 효과 없는데 있다고 판단 (False Positive)
- **2종 오류**: 효과 있는데 없다고 판단 (False Negative)

## 3과목 — 빅데이터 모델링 (전반)

### 지도학습 vs 비지도학습
- **지도학습**: 라벨(정답) 있음
- **비지도학습**: 라벨 없음

### 분류 vs 회귀
- **분류(Classification)**: 출력이 카테고리 (스팸/아님, 암/정상)
- **회귀(Regression)**: 출력이 연속적인 숫자 (집값, 온도)

### 선형회귀
- 데이터에 가장 잘 맞는 직선을 찾아 값 예측
- 오차 기준: **MSE(평균제곱오차)** 최소화
- 제곱하는 이유: 양수/음수 상쇄 방지

### 로지스틱 회귀
- 이름에 '회귀'가 붙지만 **분류** 문제에 사용
- 출력: 0~1 사이 확률 → 0.5 기준으로 분류

### SVM (Support Vector Machine)
- 두 클래스를 나누는 **마진(Margin)을 최대화**하는 경계선 찾기
- **서포트 벡터**: 경계선에 가장 가까운 데이터 포인트들

### 결정트리 (Decision Tree)
- 각 노드에서 질문 → Yes/No로 가지 분기 → 잎 노드에서 결론
- 단점: 너무 깊으면 **과적합(Overfitting)**
- 해결: **가지치기(Pruning)**

### 랜덤포레스트 (Random Forest)
- 결정트리 여러 개 → **다수결**로 결론
- **앙상블(Ensemble)** 방법 중 **배깅(Bagging)** 방식
  - 배깅: 트리들이 독립적으로 동시에 학습

### 부스팅 (Boosting)
- 트리를 **순서대로** 생성
- 앞 트리가 틀린 데이터에 다음 트리가 집중 → 약점 보완
- 대표: **Gradient Boosting**, **XGBoost**

### 배깅 vs 부스팅
| | 배깅 | 부스팅 |
|---|---|---|
| 학습 순서 | 동시 (독립적) | 순서대로 |
| 방식 | 다수결 | 앞 트리 오류 보완 |
| 대표 | 랜덤포레스트 | XGBoost, Gradient Boosting |

### 비지도학습 — 클러스터링

#### K-Means
- K개의 중심점 랜덤 설정 → 가까운 데이터끼리 묶음 → 중심 재계산 → 반복
- 단점: K 미리 정해야 함, 초기값에 따라 결과 달라짐, 원형 모양만 잘 잡음

#### DBSCAN
- **밀도** 기반 클러스터링
- 장점: K 미리 안 정해도 됨, 복잡한 모양 클러스터도 잡음

### 연관분석 (Association Rule)
- "A 사면 B도 산다" 패턴 찾기 (장바구니 분석)
- 3대 지표:
  - **지지도(Support)**: A+B 같이 구매한 비율
  - **신뢰도(Confidence)**: A 산 사람 중 B도 산 비율
  - **향상도(Lift)**: 실제 연관성 여부 (1보다 크면 연관 있음)

### 딥러닝 기초
- 인공신경망: 뇌의 뉴런 구조 모방
  - 뉴런 → 노드, 시냅스 → 가중치(Weight)
  - 활성화 함수: 신호가 충분히 강하면 다음 노드로 전달
- 구조: **입력층 → 은닉층 → 출력층**
- 은닉층이 여러 겹 = **딥러닝 (Deep Learning)**
- 신경망은 그물망 구조 — 모든 노드가 다음 층의 모든 노드에 연결

### 학습 과정
- **순전파(Forward)**: 입력 넣어서 예측
- **역전파(Backward)**: 오차를 뒤에서 앞으로 전달하며 가중치 조정
- **학습률(Learning Rate)**: 가중치 조정 폭 (너무 크면 튀고, 너무 작으면 느림)

### 활성화 함수
- **ReLU**: 0보다 작으면 0, 크면 그대로. 은닉층에서 주로 사용
- **Sigmoid**: 출력을 0~1 확률로 압축. 이진 분류 출력층 (스팸/아님)
- **Softmax**: 여러 클래스의 확률 출력, 합이 1. 다중 분류 출력층 (개/고양이/새)

### 과적합 방지
- **드롭아웃(Dropout)**: 학습 중 랜덤으로 노드를 꺼서 특정 노드 조합에 의존하는 것 방지
  - 그물망 구조라 노드 꺼도 고장 안 남
- 그 외: 데이터 많이 쓰기, 층/노드 수 줄이기, 조기 종료(Early Stopping)
