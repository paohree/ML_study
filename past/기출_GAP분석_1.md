# 기출 GAP 분석 — 기출문제 PDF 전체 분독 결과

기출문제1, 기출문제2 PDF 전체 (1~4과목) 분석.
필수노트에 없거나 약한 개념들만 추출함.
세션이 바뀌어도 여기서 이어서 수업 가능.

## 1과목 GAP

**Veracity (신뢰성)**
5V 중 Veracity = 데이터의 신뢰성/정확성.
함정: "유용한 정보 추출" = Value. Veracity가 아님.

**MapReduce 흐름**
Map → 셔플/소트 → Reduce. 셔플/소트 빠뜨리면 틀린 답.

**DW 4특성**
주제지향성 / 통합성 / 시계열성 / 비휘발성.
함정: "실시간성"은 DW 특성 아님 (OLTP 쪽).

**데이터 거버넌스 3요소**
원칙(Principle) / 조직(Organization) / 프로세스(Process).
IT인프라는 포함 안 됨.

**변수 척도**
| 척도 | 핵심 | 예 |
|---|---|---|
| 명목 | 분류만, 순서/크기 없음 | 혈액형, 성별 |
| 순서(서열) | 순서 있음, 간격 불일치 | 학점, 만족도 |
| 등간 | 간격 일정, 절대 0 없음 | 섭씨온도, 시험점수 |
| 비율 | 간격 일정 + 절대 0 있음 | 키, 몸무게, 매출 |

함정: 섭씨온도 = 등간 (0도가 온도 없음이 아님). 비율척도라고 하면 틀림.

**빅분기 분석방법론 5단계**
분석 기획 → 데이터 준비 → 데이터 분석 → 시스템 구현 → 평가 및 전개.
분석 활용 계획은 처음 단계부터 수립 (모델링 후가 아님).

**비식별화 기술**
가명처리 / 총계처리 / 데이터 범주화 / 데이터 마스킹 / 랜덤라운딩 / 제어라운딩.
- 랜덤라운딩: 임의로 올림/내림
- 제어라운딩: 합계가 동일하게 유지되도록 라운딩

**MongoDB**
오토샤딩(Auto-Sharding) 지원.

**빅데이터 플랫폼 계층**
소프트웨어(앱/데이터수집분석) / 플랫폼(작업관리/자원할당) / 인프라스트럭처(네트워크/스토리지)

**분석 준비도 항목**
분석업무파악 / 인력및조직 / 분석기법 / 분석데이터 / 분석문화 / IT인프라.
함정: "조직규모"는 분석 준비도 항목 아님.

## 2과목 GAP

**z-score 계산**
z = (x - μ) / σ
예: 평균 60, 표준편차 10, x=70 → z = (70-60)/10 = 1

**summary 해석 (박스플롯 관련)**
이상치 기준: Q3 + 1.5×IQR 초과. Max값 자체가 이상치가 아닐 수 있음.
함정: "max보다 큰 이상치가 있다" → Wind Max=20.7이 바로 IQR 기준 이상치.

**왜도와 이상치**
데이터: 2,3,4,5,6,7,8,9,10,50
→ 50이 이상치, 오른쪽 꼬리(양의 왜도).
함정: "이상치가 없음을 확인할 수 있다" → 틀린 답. IQR 기준으로 이상치 존재.

**가설검정 신뢰수준**
항상 95%가 아님. 상황에 따라 90%, 99% 등 선택 가능.
함정: "가설검정은 항상 95%의 신뢰수준에서 수행된다" → 틀린 답.

**베이즈 정리 계산**
P(A|B) = P(B|A)×P(A) / [P(B|A)×P(A) + P(B|Aᶜ)×P(Aᶜ)]
예: 도시 내 여성 40%, 180cm 이상 남성 15%, 여성 2.5%
→ P(여성|180이상) = 0.025×0.4 / (0.025×0.4 + 0.15×0.6) = 0.01/0.10 = 0.10 = 10%

**인코딩 종류 4가지**
| 종류 | 설명 |
|---|---|
| One-Hot Encoding | 범주를 이진 열로 변환 (고양이=[1,0], 개=[0,1]) |
| Label Encoding | 범주를 고유 숫자로 매핑 |
| Ordinal Encoding | 범주를 순서에 따라 숫자 할당 |
| Binary Encoding | 범주를 이진 코드로 표현, One-Hot보다 빠름 |

**정준상관분석 (CCA)**
다수의 독립변수 ↔ 다수의 종속변수 상관관계 분석.
변수 그룹 간 최대 상관관계를 갖는 선형 조합 찾기.

**표본분산과 카이제곱분포**
정규분포 모집단에서 표본분산 s²은 자유도 n-1의 카이제곱 분포를 따름.
- S1 = (1/n)Σ(Xi-X̄)² → 편향 추정량
- S2 = (1/(n-1))Σ(Xi-X̄)² → 불편추정량 (편향=0)

**상관계수 4종**
| 상황 | 계수 |
|---|---|
| 연속형 ↔ 연속형 (선형) | 피어슨 |
| 연속형 ↔ 연속형 (비선형/이상치) | 스피어만 |
| 이진(0/1) ↔ 연속형 | 포인트-비세리얼 |
| 범주형 ↔ 범주형 | 크라메르의 V |

**정준상관 vs 편상관**
- 정준상관: 다수 독립 ↔ 다수 종속
- 편상관: 다른 변수 영향 통제 후 두 변수 순수 관계

**공분산 행렬 해석**
- 대각선: 각 변수의 분산
- 비대각선: 공분산 (음수면 음의 상관)
- 상관계수 = 공분산 / (표준편차1 × 표준편차2)
- 함정: X1과 X3의 상관계수 = 0이 아님 (공분산이 0일 때만 0)

**검정 종류**
| 상황 | 검정 |
|---|---|
| 두 집단 평균 비교 | t검정 |
| 세 집단 이상 평균 비교 | ANOVA(F검정) |
| 범주형 변수 간 독립성/적합도 | 카이제곱 검정 |
| 정규분포 가정 없을 때 | 비모수 검정 |

함정: 비모수 검정이 모수 검정보다 검정력이 높다 → 틀린 답.

**중심극한정리**
표본 크기 ≥ 30이면 모집단 분포 관계없이 표본평균이 정규분포에 근사.
함정: "표본 크기와 관계없이" → 틀린 답.

**SMOTE**
소수 클래스 데이터를 인위적으로 생성하는 오버샘플링 기법.
불균형 데이터: 계층별(Stratified) k-fold 교차검증도 활용 가능.
함정: "소수 클래스를 언더샘플링하여 해결한다" → 틀린 답 (오버샘플링).

**변동계수 (CV)**
표준편차 ÷ 평균. 단위가 다른 자료 비교할 때 사용.

**상대위험도 & 승산비 계산**
- 상대위험도(RR) = 신규 기술 불량률 / 기존 기술 불량률
- 승산비(OR) = (신규 승산) / (기존 승산)
  - 승산 = 불량/(전체-불량)

**결측값 대체 방법**
- 평균/중앙값 단일값 대체: 분산이 작아짐 (편향 증가)
- 여러 번 대체: 편향 작아짐
- 난수 추가 대체: 원래 분산 유지에 도움
- 회귀분석 결과로 대체: 변수 간의 관계 보존에 도움됨
  함정: "회귀분석으로 대체하면 변수 관계가 약해진다" → 틀린 답

**배치(batch) 크기**
배치 크기는 훈련 속도뿐만 아니라 성능에도 영향을 줌.
함정: "배치 크기는 속도에만 영향을 주지 성능에는 영향을 주지 않는다" → 틀린 답.

## 3과목 GAP

**Lasso / Ridge / Elastic Net**
- Ridge(L2): 가중치 제곱합 최소화 → 계수 0에 가깝게 줄임 (완전 0은 안 됨)
- Lasso(L1): 가중치 절대값 합 최소화 → 일부 계수 아예 0으로 만듦 (변수 선택)
- Elastic Net: L1 + L2 결합
- 후진제거법(Backward Elimination): 가장 영향 적은 변수 순서대로 제거

변수가 많고 의미있는 것이 일부일 때 → Lasso 유리.
함정: "Lasso는 계수를 0에 가깝게만 줄인다" → 틀린 답.

**나이브 베이즈**
사전확률 + 우도 → 사후확률 계산.
모든 특성이 서로 독립이라고 가정.
함정: "사전확률과 사후확률로 우도를 계산한다" → 틀린 답.

**k-fold 교차검증**
- k-1개 학습용, 1개 검증용. k번 반복.
- 행 단위로 나누는 게 일반적 (열 단위 아님).
- 홀드아웃보다 신뢰도 높지만 느림. k 클수록 신뢰도 높음.
함정: "홀드아웃보다 계산이 빠르다" → 틀린 답.

**계층적 군집분석**
K 미리 안 정해도 됨. 결과를 덴드로그램으로 표현. 사용자가 어디서 자를지 결정.
함정: "사전에 군집 수를 지정한다" → 틀린 답.

**K-평균 최적 k 탐색: 엘보우 메소드**
집단 내 제곱합(SSW)이 급격히 감소하다가 서서히 감소하는 지점(팔꿈치) = 최적 k.

**PCA (주성분 분석)**
- 첫 번째 주성분 = 가장 큰 분산 방향
- 주성분들 간 상관관계 없음 (직교)
- 변수들 간 상관관계가 없어도 PCA는 가능
- 함정: "정규분포를 가정한다" → PCA는 정규분포 가정 필수가 아님
- PCA 결과 해석: 누적기여율에서 차 계산
  예: Comp.3의 기여율 = 누적(3) - 누적(2) = 0.9566 - 0.8675 = 0.0891 = 8.91%

**SVD (특이값 분해)**
행렬을 특이벡터(U, V)와 특이값(Σ)으로 분해. 직사각형 행렬도 가능.
함정: "정방행렬에만 적용 가능하다" → 틀린 답.

**NLP 모델 구분**
- BERT: 양방향 문맥 파악. 함정: "단방향" → 틀린 답.
- GPT: 대용량 텍스트 사전학습 후 fine-tuning.
- Word2Vec: 단어를 벡터로 변환 (의미 공간 표현).
- YOLO: NLP 아님! 실시간 이미지 객체탐지(You Only Look Once).

**Transformer vs LSTM**
- LSTM: 순서대로 읽음. Forget Gate로 기억 제어. 긴 문장에서 앞 내용 망각 문제.
- Transformer: 전체 문장 한번에 봄. Self-Attention 메커니즘.
  구성요소: Self-Attention / Multi-Head Attention / Positional Encoding / Feedforward NN
  함정: "Forget Gate는 Transformer 구성요소다" → 틀린 답. LSTM 구성요소.

**Seq2seq (시퀀스 투 시퀀스)**
- Encoder: 입력 시퀀스 → 컨텍스트 벡터(Context Vector, 고정 크기)
- Decoder: 컨텍스트 벡터 → 출력 시퀀스
- 활용: 기계 번역, 자동 요약, 질문 응답

**역전파 (Backpropagation)**
출력부터 반대방향으로 편미분하면서 가중치를 조정.
(ㄱ)=편미분, (ㄴ)=가중치

**신경망 출력값 계산**
출력 = Σ(노드값 × 가중치) + 편향, 그 후 활성화함수 적용.
예: 0.2×0.4 + 0.1×0.5 + 0.2 = 0.33
ReLU: 0 미만이면 0으로 변환.

**Vanishing Gradient (기울기 소실)**
깊은 층에서 초기 층으로 갈수록 그래디언트가 점점 작아지는 현상.
- Sigmoid 사용 시 발생 (완화가 아닌 원인).
- ReLU로 해결.
- 그래디언트 클리핑: Gradient Explosion(폭발) 방지용.
함정: "시그모이드 사용하면 완화된다" → 틀린 답.

**경사하강법**
- SGD(확률적 경사하강법): 무작위 선택 데이터의 그래디언트로 업데이트 (전체 아님)
- 모멘텀: 관성으로 수렴 속도 개선, 지역 최소값 탈출
- 학습률 너무 크면 최적점 넘어서거나 발산
함정: "SGD는 전체 데이터셋의 편미분을 사용한다" → 틀린 답.

**의사결정나무**
- 비모수적(non-parametric): 데이터 분포 가정 없음
- 정지규칙: 최대 깊이, 최소 샘플 수, 정보이득 최소 향상치
  함정: "가지에 남은 개수가 같으면 멈춘다" → 틀린 답
- 교호작용 파악 어려움 (복잡한 변수 조합 효과)
함정: "정규성 가정이 필요하다" → 틀린 답.

**SVM**
초매개변수(C, 커널 함수 매개변수) 최적화 필요.
함정: "초매개변수 최적화는 필요 없다" → 틀린 답.

**부스팅 종류**
| 알고리즘 | 핵심 |
|---|---|
| AdaBoost | 오류 데이터에 더 큰 가중치 부여, 순차 학습 |
| GBM | 경사하강법으로 오류 최소화, 순차 모델 구축 |
| XGBoost | GBM보다 빠름 (병렬처리, 가지치기, 정규화, 결측치 자동처리) |
| LightGBM | leaf-wise-node 방식으로 분기 |

함정: "XGBoost는 GBM보다 속도가 늦다" → 틀린 답.

**배깅 vs 부스팅**
- 배깅: 모델 간 독립적 → 병렬처리 가능. 랜덤포레스트가 배깅의 하나.
- 부스팅: 잘못 분류된 데이터에 가중치 → 순차 처리.

**랜덤포레스트**
나무 개수 많을수록 성능 향상되나 어느 수준 이상에서는 효과 미미.
함정: "모델에 사용된 모형의 개수는 많을수록 좋다" → 틀린 답.

**MANOVA**
독립변수 1개 이상 + 종속변수 여러 개인 경우 사용.
함정: "독립변수가 여러 개이고 종속변수가 하나인 경우" → ANOVA, MANOVA 아님.

**스태킹(Stacking)**
여러 모델의 예측 결과를 새로운 학습 데이터로 사용하는 앙상블.

**SOM (자기조직화지도)**
비지도학습으로 고차원 데이터를 저차원(주로 2D) 뉴런 그리드에 매핑.

**소셜미디어 데이터 분석**
텍스트 마이닝 / 워드 클라우드 / 네트워크 분석.
함정: 맵리듀스 → 대규모 데이터 처리 프로그래밍 모델, 소셜미디어 분석 방법 아님.

**잔차 자유도**
자유도 = n - p (n=관측 수, p=파라미터 수, 절편 포함).
함정: "잔차의 자유도는 항상 n-1이다" → 틀린 답. 단순회귀는 n-2, 다중회귀는 n-p.

**ANOVA 테이블 해석**
- SSR(회귀 제곱합), SSE(잔차 제곱합), SST=SSR+SSE
- F값 = MSR/MSE = (SSR/자유도1) / (SSE/자유도2)

## 4과목 GAP

**혼동행렬 계산**
T/F = 예측이 맞냐 틀리냐 / P/N = 뭐라고 예측했냐

|  | 예측 Positive | 예측 Negative |
|---|---|---|
| 실제 Positive | TP | FN |
| 실제 Negative | FP | TN |

- Accuracy = (TP + TN) / 전체
- Recall(민감도/재현율) = TP / (TP + FN)
- Precision(정밀도) = TP / (TP + FP)
- Specificity(특이도) = TN / (TN + FP)
- F1 = 2 × (Precision × Recall) / (Precision + Recall)

예시 계산 (TP=80, FP=10, FN=20, TN=90):
- Accuracy = 170/200 = 0.85
- Recall = 80/100 = 0.80
- Specificity = 90/100 = 0.90
- Precision = 80/90 = 0.89

**ROC 곡선 & AUC**
- X축: 1-특이도 (FPR = FP Rate)
- Y축: 민감도 (TPR = TP Rate = Recall)
- AUC 1에 가까울수록 좋음, 0.5 = 무작위
- 함정: "X축=특이도" → 틀린 답. "AUC 작을수록 좋다" → 틀린 답.

**평가지표 사용 상황**
- 암 진단 (놓치면 안 됨) → Recall 중시
- 스팸 필터 (정상 메일 스팸 처리 안 됨) → Precision 중시
- 회귀 모델 (전기사용량 예측 등) → MAE/MSE/RMSE/R² 사용
  함정: F1 score는 회귀 모델에 사용 불가 (분류 전용)

**회귀 평가지표**
- MAE: 절대값 평균. 이상치에 덜 민감.
- MSE: 제곱 평균. 이상치에 가장 민감.
- RMSE: MSE에 루트. 단위 복원.
- R²(결정계수): 0~1 사이. 모델이 데이터 변동 설명하는 비율.
  다변량이면 Adjusted R² 사용.
  함정: "종속변수가 독립변수를 설명하는 비율" → 반대. 독립→종속.

**편향(Bias) - 분산(Variance) 트레이드오프**
- 단순 모델: 편향 높음, 분산 낮음 (언더피팅)
- 복잡 모델: 편향 낮음, 분산 높음 (오버피팅)
- 좋은 모델: 편향 낮고 분산 낮음
함정: "단순한 모델은 편향이 낮고 분산이 높다" → 틀린 답.

**시각화 종류**
| 상황 | 그래프 |
|---|---|
| 분포 (연속형) | 히스토그램 |
| 이상치 확인 | 박스플롯 |
| 두 변수 관계 | 산점도 |
| 시간 흐름 | 선그래프 |
| 그룹 비교 (범주형) | 막대그래프 |
| 전체 중 비율 | 원형 차트 |
| 여러 변수 동시 비교 | 레이더 차트(방사형) |
| 세 변수 표현 (x, y + 크기) | 버블 차트 |
| 범주형 두 변수 비율+관계 | 모자이크 플롯 |

함정: 히스토그램은 범주형 시각화 아님 (연속형 전용).
박스플롯: 95% 신뢰구간 알 수 없음.

**비교시각화**
여러 변수 또는 데이터 집합 간 비교. 특징/차이점/유사점 한눈에 파악.
레이더 차트, 나란히 막대그래프 등.

**지도형 시각화 (인포그래픽)**
- 지도형: 지역별 분포/현황
- 타임라인형: 시간 흐름에 따른 변화
- 비교분석형: 항목 간 비교
- 도표형: 표나 차트로 수치 표현
- 스토리텔링형: 이야기 흐름으로 전달

**카토그램 (Cartogram)**
지리적 면적이 아닌 특정 통계값(인구, GDP 등)에 따라 지역 크기를 조정하여 표시하는 지도.
타일격자지역(Grid Cartogram): 모든 지역을 동일한 크기의 타일로 표현.

**텍스트 마이닝 전처리**
- Stop Words Removal: 의미 낮은 단어 제거 (불용어)
- Stemming: 단어를 어근 형태로 변환
- Tokenization: 문장/단어 단위로 분리
- POS 태깅: 각 단어의 품사 식별 및 태깅 (단어의 뜻이 아닌 품사)
함정: "POS 태깅은 단어의 뜻을 판단한다" → 틀린 답. 품사 식별.

**거리 종류**
- 민코우스키: 맨하탄 + 유클리디안을 일반화한 거리 (r=1이면 맨하탄, r=2이면 유클리디안)
- 마할라노비스: 데이터의 공분산 구조를 고려한 거리
- 유클리드: 직선 거리
- 맨해튼: 격자 거리 (가로+세로)

**차원 축소 목적**
노이즈 제거 / 특징 추출 / 계산 효율 향상.
함정: "설명력 증가" → 틀린 답. 차원 축소 과정에서 일부 정보 손실 불가피.

**로지스틱 회귀 vs 선형 회귀**
- 로지스틱: 종속변수 범주형. 잔차 정규성 가정 아님. 로그 오즈(log odds)가 선형적으로 변한다고 가정.
- 선형: 잔차 정규성 가정. LSE로 계수 추정 시 불편추정량.
함정: "선형+로지스틱 회귀 모두 잔차 정규성을 가정한다" → 틀린 답.

## 계산 문제 정리 (시험 자주 출제)

**z-score**: z = (x - μ) / σ

**베이즈 정리**: P(A|B) = P(B|A)P(A) / [P(B|A)P(A) + P(B|Aᶜ)P(Aᶜ)]

**향상도(Lift)**: P(A∩B) / (P(A)×P(B))
예: P(사과)=0.8, P(달걀,우유)=0.6, P(사과∩달걀,우유)=0.4
→ 향상도 = 0.4/(0.8×0.6) = 0.833

**신경망 출력**: Σ(노드값 × 가중치) + 편향 → 활성화함수 적용
예: 0.2×0.4 + 0.1×0.5 + 0.2 = 0.33

**PCA 기여율**: 특정 주성분 기여율 = 누적기여율(n) - 누적기여율(n-1)

**상대위험도(RR)**: 신규 불량률 / 기존 불량률
**승산비(OR)**: (신규 불량/신규 정상) / (기존 불량/기존 정상)

**1종/2종 오류 확률 계산**:
- 1종 오류: H0가 참인데 기각할 확률
- 2종 오류: H0가 거짓인데 기각 못할 확률 → H1이 참일 때 기각역 바깥 확률
