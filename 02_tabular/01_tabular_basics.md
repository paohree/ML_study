# 테뷸러 데이터 기초

## 테뷸러 데이터란?

행과 열로 이루어진 표 형태 데이터다.  
엑셀, CSV, 데이터베이스 테이블이 대표적이다.

예시 컬럼:
- 나이
- 성별
- 지역
- 구매 금액
- 가입 기간

이런 데이터에서는 각 컬럼의 의미를 해석하고 전처리하는 능력이 중요하다.

## EDA

EDA는 `Exploratory Data Analysis`의 줄임말이다.  
데이터를 본격적으로 학습시키기 전에 상태를 파악하는 과정이다.

주로 보는 것:
- 행 수, 열 수
- 결측치 개수
- 숫자형 / 범주형 컬럼 구분
- target 분포
- 이상치 존재 여부
- 컬럼 간 관계

한 줄 요약:
- 모델보다 먼저 데이터를 이해하는 단계

## 결측치 처리

결측치는 값이 비어 있는 상태다.

처리 방법:
- 삭제
- 평균/중앙값으로 채우기
- 최빈값으로 채우기
- 별도 값으로 채우기

중요한 점:
- 결측 자체가 의미일 수도 있다
- 무작정 삭제하면 정보가 많이 사라질 수 있다

## 범주형 인코딩

모델은 보통 문자열을 직접 이해하지 못하므로 숫자로 바꿔야 한다.

### One-Hot Encoding

범주별로 컬럼을 따로 만드는 방식

예:
- color = red, blue
- color_red, color_blue 컬럼 생성

장점:
- 범주 간 크기 관계를 강제로 만들지 않음

단점:
- 범주 수가 많으면 컬럼이 너무 늘어남

### Label Encoding

범주를 숫자 하나로 바꾸는 방식

예:
- red -> 0
- blue -> 1
- green -> 2

주의:
- 숫자 크기에 의미가 없는데 순서가 생긴 것처럼 보일 수 있다

## 스케일링

숫자 feature들의 범위를 맞추는 과정이다.

### Standardization

평균 0, 표준편차 1 근처로 맞춤

### Normalization

값 범위를 0~1 등으로 줄임

언제 중요한가?
- 선형 모델
- 거리 기반 모델
- 신경망

언제 덜 중요한가?
- 트리 기반 모델

## Feature Engineering

기존 컬럼을 바탕으로 더 유용한 feature를 만드는 작업이다.

예시:
- 생년월일 -> 나이
- 가입일 -> 가입 후 경과일
- 총구매액 / 구매횟수 -> 평균 구매액

실전에서는 모델 자체보다 feature engineering이 더 중요할 때가 많다.

## 이상치 처리

다른 데이터와 지나치게 다른 값을 뜻한다.

예시:
- 나이가 999
- 매출이 비정상적으로 큼

주의:
- 진짜 오류인지
- 실제로 드문 정상 데이터인지

둘을 구분해야 한다.

## Data Leakage

예측 시점에는 알 수 없는 정보가 학습에 들어간 상태다.

예시:
- 대출 연체 예측인데, 연체 발생 후에 기록된 정보를 feature로 사용

이 문제는 점수를 가짜로 높게 만든다.  
입문 단계부터 강하게 경계해야 한다.

## 클래스 불균형

한쪽 클래스가 지나치게 많은 상태다.

예:
- 정상 95%, 사기 5%

이 경우 Accuracy만 보면 착시가 생긴다.  
Precision, Recall, F1, ROC-AUC를 함께 봐야 한다.

## 입문자가 잡아야 할 감각

- 테뷸러 ML은 "모델 고르기"보다 "데이터 정리"가 더 중요하다
- 문자열, 결측치, 이상치 처리를 이해해야 한다
- 데이터의 생성 시점과 컬럼 의미를 꼭 확인해야 한다
- 점수가 잘 나와도 leakage 가능성을 먼저 의심해야 한다
