# ML 기초

## 머신러닝이란?

머신러닝은 사람이 규칙을 하나하나 직접 쓰는 대신, 데이터를 보고 규칙을 학습하게 만드는 방법이다.

- 입력값: `feature`
- 맞히고 싶은 값: `target` 또는 `label`
- 데이터 한 줄: `sample` 또는 `row`

예시:
- 집 크기, 방 개수로 집값 예측 -> 회귀
- 고객 정보로 이탈 여부 예측 -> 분류

## 머신러닝의 큰 분류

### 1. 지도학습

정답이 있는 데이터를 가지고 학습한다.

- 입력: feature
- 출력: label
- 대표 작업: 분류, 회귀

초보자가 가장 먼저 공부해야 하는 영역이다.

### 2. 비지도학습

정답 없이 데이터의 구조를 찾는다.

- 군집화
- 차원 축소

입문에서는 우선순위가 높지 않다.

### 3. 강화학습

행동에 대한 보상을 최대화하는 방향으로 학습한다.

- 게임
- 로봇 제어
- 의사결정 문제

테뷸러 입문과는 거리가 있다.

## 분류와 회귀

### 분류

결과가 범주형일 때 사용한다.

- 스팸 메일인가 아닌가
- 고객이 탈퇴할까 말까
- 대출이 승인될까 아닐까

대표 지표:
- Accuracy
- Precision
- Recall
- F1 score
- ROC-AUC

### 회귀

결과가 숫자일 때 사용한다.

- 매출 예측
- 가격 예측
- 수요 예측

대표 지표:
- MAE
- MSE
- RMSE
- R2

## 과적합과 과소적합

### 과적합

훈련 데이터는 너무 잘 외웠는데, 새로운 데이터에서는 성능이 떨어지는 상태다.

징후:
- train 점수는 높음
- validation/test 점수는 낮음

원인:
- 모델이 너무 복잡함
- 데이터가 적음
- 불필요한 feature가 많음

### 과소적합

모델이 너무 단순해서 훈련 데이터도 잘 못 맞히는 상태다.

징후:
- train 점수도 낮음
- validation/test 점수도 낮음

## 편향-분산 트레이드오프

- 편향이 크다: 모델이 너무 단순해서 패턴을 잘 못 배움
- 분산이 크다: 데이터에 너무 민감해서 흔들림이 큼

실전에서는 이 둘의 균형을 맞추는 게 중요하다.

## 데이터 분리

### Train

모델을 학습시키는 데이터

### Validation

하이퍼파라미터를 조정하거나 모델 선택에 쓰는 데이터

### Test

마지막에 한 번만 성능 확인하는 데이터

중요:
- test 데이터는 여러 번 들여다보면 안 된다
- validation 정보가 train에 새어 들어가면 `data leakage`가 생긴다

## 교차검증

데이터를 여러 번 나눠서 검증하는 방식이다.

왜 쓰나?
- 데이터가 적을 때 더 안정적으로 평가 가능
- 한 번 나눈 결과에 덜 흔들림

대표 방식:
- K-Fold Cross Validation

## 평가지표

### Accuracy

전체 중 맞춘 비율.  
클래스 불균형이 심하면 착시가 생길 수 있다.

### Precision

양성이라고 예측한 것 중 진짜 양성 비율

### Recall

진짜 양성 중에서 양성이라고 잘 찾아낸 비율

### F1 Score

Precision과 Recall의 균형을 본다.

### ROC-AUC

분류 모델의 전반적인 구분 능력을 본다.

### RMSE

예측값과 실제값 차이를 숫자로 보는 대표 회귀 지표

## 입문자가 기억할 핵심

- 처음에는 "문제가 분류인지 회귀인지"부터 구분
- 성능보다 "왜 이 모델을 쓰는지"를 먼저 이해
- 한 번의 점수보다 데이터 분리와 검증 방식이 더 중요
- 테뷸러 데이터에서는 딥러닝보다 트리 기반 모델이 먼저다
