## 학습 플랫폼

단순 Claude 수업 도구를 넘어, **멀티에이전트 기반 자동화 학습 플랫폼**으로 구성되어 있다.
수업 / 평가 / 기록을 각각 독립된 AI 에이전트가 담당하고, Claude Code Harness가 세션 생명주기를 자동화한다.

### 에이전트 구조

세 에이전트는 역할이 분리되어 있으며, `진행상황.md`를 공유 문서로 삼아 통신한다.

```
┌─────────────────────────────────────────────────────┐
│  수업 AI  (Claude — 메인 세션)                         │
│                                                     │
│  · 진행상황.md 읽고 오늘 수업 내용 결정                     │
│  · 소크라테스식 대화로 수업 진행                           │
│    ("왜 이런 문제가 생길까?" → 학생이 답 → 설명)            │
│  · 수업 단위 완료 시 평가 AI 호출                         │
└────────────────────┬────────────────────────────────┘
                     │ 수업 내용 요약 + 학생 반응 전달
                     ▼
┌─────────────────────────────────────────────────────┐
│  평가 AI  (Claude — 서브에이전트, 태스크 단위 실행)         │
│                                                     │
│  · 수업 내용 기반 퀴즈 3~5문제 생성 (필기시험 스타일)          │
│  · 학생 답변 채점 + 오답 패턴 분석                         │
│  · 진행상황.md 강점/약점 업데이트                          │ 
│  · session_log.md 생성 (기록 AI용 인계 문서)             │
│  · 다음 수업 방향 수업 AI에게 반환                         │
└────────────────────┬────────────────────────────────┘
                     │ 세션 종료 시 (Stop hook)
                     ▼
┌─────────────────────────────────────────────────────┐
│  기록 AI  (OpenAI Codex CLI — 별도 프로세스)            │
│                                                     │
│  · session_log.md 읽기                               │
│  · 루트의 과거 YYMMDD.md → past/ 자동 이동               │
│  · 오늘 날짜로 YYMMDD.md 생성                           │
│  · session_log.md 삭제 (처리 완료)                     │
│  · 폴더 정합성 유지 (루트엔 오늘 날짜 md만 존재)             │
└─────────────────────────────────────────────────────┘
```

### 에이전트 간 공유 문서

| 문서 | 역할 |
|---|---|
| `진행상황.md` | 수업 AI가 읽고, 평가 AI가 업데이트하는 핵심 상태 문서 |
| `.claude/session_log.md` | 평가 AI → 기록 AI 인계 문서. 세션 요약, 퀴즈 결과, 약점 포함 |
| `YYMMDD.md` | 기록 AI가 생성하는 날짜별 학습 노트 |

### 자동화 (Claude Code Harness)

Claude Code의 `hooks` 기능으로 세션 생명주기를 자동화한다. 공부하지 않는 날에는 Claude 세션을 열지 않으면 에이전트가 전혀 실행되지 않아 불필요한 토큰 소비가 없다.

| 시점 | 동작 | 구현 |
|---|---|---|
| 세션 종료 | 기록 AI (Codex) 실행 | Stop hook |
| 세션 종료 | 변동사항 git commit & push | Stop hook |
| 매일 12시 | 변동사항 있으면 git push | cron |
| 매일 17시 | 변동사항 있으면 git push | cron |

에이전트 역할 정의는 [`agents/`](agents/) 폴더에서 관리한다.
